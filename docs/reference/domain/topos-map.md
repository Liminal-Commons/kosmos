## Ontological Layers

### Foundation Layer (Identity & Governance)

```
hypostasis (who exists)
    │
    └──▶ politeia (how they relate)
             │
             └──▶ Oikoss, memberships, attainments
```

**hypostasis** establishes *who* — prosopa, parousiai, keyrings.
**politeia** establishes *how they relate* — oikoi, memberships, delegated authority.

### Embodiment Layer (Body & Mind)

```
soma (how we sense/act)
    │
    └──▶ nous (how we understand)
             │
             └──▶ Body-schema, theoria, journeys
```

**soma** provides the body — channels of attention, substrates of capability.
**nous** provides the mind — surfacing, crystallizing, questioning.

### Creation Layer (Making & Generating)

```
demiurge (composition)
    │
    └──▶ manteia (generation)
             │
             └──▶ Typed creation, governed inference
```

**demiurge** composes from definitions — the single act of creation.
**manteia** generates with governance — LLM output constrained by schema and policy.

### Dwelling Layer (Presence & Publication)

```
oikos (intimate presence)
    │
    └──▶ ekdosis (public release)
             │
             └──▶ Sessions, notes, releases
```

**oikos** provides dwelling — structured presence in an oikos.
**ekdosis** provides publication — moving from private to public.

### Actuality Layer (Intent & Reality)

```
dynamis (bridging intent/actuality)
    │
    └──▶ dokimasia (validating fitness)
             │
             └──▶ Reconciliation, validation gates
```

**dynamis** reconciles — sensing actuality, manifesting intent.
**dokimasia** validates — ensuring authorization implies validity.

### Boundary Layer (Entry & Exit)

```
propylon (entry)
    │
    └──▶ thyra (phasis)
             │
             └──▶ Links, streams, commitment boundary
```

**propylon** manages entry — invitation links, authentication, session tokens.
**thyra** manages phasis — streams, rendering, the send moment.

### Coordination Layer (Work & Transport)

```
ergon (work coordination)
    │
    └──▶ aither (transport)
             │
             └──▶ Pragma signals, syndesmos connections
```

**ergon** coordinates work — pragma entities signal gaps to capability.
**aither** provides transport — P2P channels without semantic knowledge.

### Gathering Layer (Space & Infrastructure)

```
agora (spatial gathering)
    │
    └──▶ dns (network addressing)
             │
             └──▶ Territories, zones, addressability
```

**agora** provides space — territories where oikoi gather with spatial presence.
**dns** provides addressing — making the kosmos reachable from outside.

## File Structure

Each topos follows this structure:

```
genesis/{topos}/
├── DESIGN.md           # Ontological purpose, oikos context, embodiment
├── manifest.yaml       # What this topos provides (v2.1 format)
├── eide/
│   └── {topos}.yaml    # Entity definitions + attainments (entities: format)
├── desmoi/
│   └── {topos}.yaml    # Bond definitions (if any)
└── praxeis/
    └── {topos}.yaml    # Operation definitions
```

**Special cases:**
- `dns` lives under `genesis/thyra/dns/` as a sub-module
- `stoicheia-portable` contains step definitions used by interpreter
